# 📦 Project Template (General)

> 이 템플릿은 **모든 프로젝트(게임 / 디자인 / 그림 / 서비스 / 실험 등)**에 공통으로 사용하는 기준 문서이다.
> 핵심은 **"내가 느낀 문제 → 이 프로젝트로 해결하려는 시도"**의 흐름을 명확히 남기는 것이다.

### 한 줄 요약

> 이 프로젝트는 어떤 문제를 해결하려는 시도인가?

---

## 2️⃣ Problem — 내가 느낀 문제

> ❗ **중요**: 여기서 Problem은
> "프로젝트를 하며 겪은 어려움"이 아니라,
> **이 프로젝트를 시작하게 만든 감정·현실·사회적 문제**를 의미한다.

* 내가 일상에서 느낀 불편함이나 고민은 무엇이었는가?
* 어떤 순간에 이 문제를 강하게 인식했는가?
* 이 문제는 나 같은 사람(소심한 사람)에게 왜 더 크게 다가오는가?

---

## 3️⃣ Why This Project — 왜 이 프로젝트인가

* 이 문제를 **다른 방식이 아니라** 이 형태의 프로젝트로 풀고 싶었던 이유는?
* 기존에 비슷한 해결 방식의 한계는 무엇이라고 느꼈는가?

---

## 4️⃣ Project Goal — 이 프로젝트의 목표

* 이 프로젝트가 해결하고자 하는 핵심은 무엇인가?
* 사용자가 얻길 바라는 변화는 무엇인가?

> 예:
>
> * 말 걸기가 덜 무서워진다
> * 실패해도 괜찮다는 감각을 얻는다
> * 선택해본 경험 자체가 남는다

---

## 5️⃣ Solution — 내가 제안하는 해결 방식

* 이 프로젝트는 위 문제를 **어떤 방식으로 해결하려 하는가?**
* 핵심 아이디어는 무엇인가?
* 사용자가 직접 경험하게 되는 행동은 무엇인가?
